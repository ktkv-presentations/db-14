Переменные в SQL

переменные

именованные области памяти, используемые для хранения промежуточных значений, которые могут меняться во время выполнения программы или SQL-запроса

Помогают сделать запросы более динамичными

Улучшают читаемость и поддержку кода

Упрощают создание сложных логических конструкций

Типы переменных в SQL

Пользовательские переменные

@variable_name

позволяют сохранять значения, которые могут использоваться повторно в пределах текущей сессии

Для обозначения пользовательской переменной в MySQL используется префикс @

Типы переменных в SQL

Системные переменные

@@session.variable

хранят конфигурационные параметры и настройки базы данных

Типы системных переменных
Сессионные
@@session

применимы только для текущей сессии и могут быть изменены в ее пределах

Глобальные
@@global

устанавливают настройки на уровне сервера и влияют на все соединения

Типы переменных в SQL

Локальные переменные

DECLARE название тип

Локальные переменные создаются в хранимых процедурах, функциях и триггерах

Инициализация и присваивание значений

SQL поддерживает несколько способов задания значений переменных

SET

используется для установки значения переменной примитивом

SELECT INTO

позволяет присваивать переменным значения из результата запроса 


DECLARE @count INT;
SET @count = 0;
SELECT COUNT(*) INTO @count FROM employees WHERE department = 'Sales';

Использование переменных в SQL-запросах

Переменные помогают создавать динамические запросы, в которых значения могут варьироваться 


DECLARE @min_salary DECIMAL(10, 2);
SET @min_salary = 50000;
SELECT * FROM employees WHERE salary > @min_salary;

Приведение типов в SQL

процесс преобразования значения одного типа данных в другой

SELECT CAST(123 AS VARCHAR(10));

Операторы и функции приведения типов в SQL

CAST

CAST(expression AS data_type)

используется для преобразования значения одного типа данных в другой

SELECT CAST(123 AS VARCHAR(10));

CONVERT

CAST(expression AS data_type)

позволяет не только преобразовывать данные, но и задавать стиль преобразования

SELECT CONVERT(VARCHAR, GETDATE(), 101);
